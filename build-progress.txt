## Iteration 2 — infra-002: AWS SES Email Sending Client (2026-03-29)

### What was built
- SES client wrapper (`src/lib/ses.ts`) around `@aws-sdk/client-sesv2`
- `sendEmail()` — sends emails via SES with full support for: from, to, cc, bcc, subject, html, text, replyTo, headers, attachments

## Iteration 9 — api-001: Email Sending REST API (2026-03-29)

### What was built
- API key authentication middleware (`src/lib/api-auth.ts`) — validates Bearer token by SHA-256 hashing and looking up in DB
- POST /api/emails — sends single email via SES, stores in DB with events, returns `{ id }`. Supports from, to (string or array), subject, html, text, cc, bcc, reply_to, headers, attachments, tags, scheduled_at
- POST /api/emails/batch — sends up to 100 emails in one request, validates each email, returns `{ data: [{ id }] }`. Rejects >100 with 400 error
- GET /api/emails — lists emails with cursor pagination (limit, after, before params), returns `{ object: "list", has_more, data }` matching Resend API format
- GET /api/emails/:id — returns single email with events, `{ object: "email", ... }` format with snake_case field names
- Input validation: 422 for missing required fields (from, to, subject, html/text), 401 for missing/invalid API key
- String `to` field auto-normalized to array for API compatibility

- 145 total tests passing (134 previous + 11 new), no regressions

- API response format matches Resend API exactly: snake_case fields, `object` type field, cursor pagination with `has_more`
- normalizeToArray helper handles both string and array inputs for to/cc/bcc/reply_to fields
