## Iteration 2 — infra-002: AWS SES Email Sending Client (2026-03-29)

### What was built
- SES client wrapper (`src/lib/ses.ts`) around `@aws-sdk/client-sesv2`
- `sendEmail()` — sends emails via SES with full support for: from, to, cc, bcc, subject, html, text, replyTo, headers, attachments
- Input validation: rejects missing from, to, subject, or body before calling SES
- Attachment support via raw MIME message builder (multipart/mixed with base64 attachments)
- `createDomainIdentity()` — creates SES domain identity, returns DKIM tokens
- `getDomainIdentity()` — checks domain verification status and DKIM info
- `deleteDomainIdentity()` — removes a domain identity from SES
- All functions use us-east-1 region as configured

### Test results
- 16 new unit tests passing (sendEmail: 10 tests, createDomainIdentity: 2, getDomainIdentity: 1, deleteDomainIdentity: 2, attachment: 1)
- 38 total tests passing (22 schema + 16 SES), no regressions
- make check: typecheck + lint passing

### Decisions
- Used `vi.hoisted()` for mock setup to handle Vitest module hoisting with top-level SES client instantiation
- Attachment emails use raw MIME builder; simple emails use SES Simple content format
- Validation errors thrown before any SES API call to fail fast

### Files changed
- src/lib/ses.ts — SES client with sendEmail, domain identity management, MIME builder
- tests/ses.test.ts — 16 unit tests covering all functions and edge cases

## Iteration 3 — infra-003: Cloudflare DNS Client (2026-03-29)

### What was built
- Cloudflare DNS client (`src/lib/cloudflare.ts`) using Cloudflare REST API v4
- `createDNSRecord()` — creates TXT, CNAME, or MX records with optional priority
- `listDNSRecords()` — lists DNS records with optional name/type filters
- `deleteDNSRecord()` — deletes a DNS record by ID
- `autoConfigureDomain()` — creates DKIM CNAME records for all SES DKIM tokens
- Input validation: rejects missing API token, zone ID, record ID, domain, and empty DKIM tokens
- Cloudflare API error handling with descriptive error messages

### Test results
- 15 new unit tests passing (createDNSRecord: 5, listDNSRecords: 3, deleteDNSRecord: 3, autoConfigureDomain: 3, validation: 1)
- 53 total tests passing (22 schema + 16 SES + 15 Cloudflare), no regressions
- make check: typecheck + lint passing

### Decisions
- Used global `fetch` (no extra HTTP library) for Cloudflare REST API calls
- DKIM records created as CNAME type pointing to `{token}.dkim.amazonses.com` (AWS SES standard)
- `autoConfigureDomain` creates records sequentially to avoid rate limiting
- Environment variables read at call time (not module load) to support testing

### Files changed
- src/lib/cloudflare.ts — Cloudflare DNS client with CRUD operations and auto-configure
- tests/cloudflare.test.ts — 15 unit tests covering all functions and edge cases

## Iteration 4 — infra-004: S3 Storage Client (2026-03-29)

### What was built
- S3 storage client (`src/lib/s3.ts`) using `@aws-sdk/client-s3` and `@aws-sdk/s3-request-presigner`
- `uploadFile()` — uploads files to S3 bucket, returns public URL and key
- `getPresignedUrl()` — generates presigned download URLs with configurable expiry (default 3600s)
- `deleteFile()` — deletes files from S3 by key
- Input validation: rejects missing key, body, or contentType before calling S3
- Uses bucket `resend-clone-storage-699486076867` in us-east-1

### Test results
- 15 new unit tests passing (uploadFile: 7 tests, getPresignedUrl: 4 tests, deleteFile: 4 tests)
- 68 total tests passing (22 schema + 16 SES + 15 Cloudflare + 15 S3), no regressions
- make check: typecheck + lint passing

### Decisions
- Used `vi.hoisted()` for mock setup consistent with SES test pattern
- Public URL format: `https://{bucket}.s3.{region}.amazonaws.com/{key}`
- Presigned URLs use `GetObjectCommand` with configurable expiry for secure downloads
- Supports all MIME types (image/*, text/html, application/pdf, etc.)

### Files changed
- src/lib/s3.ts — S3 client with upload, presigned URL, and delete operations
- tests/s3.test.ts — 15 unit tests covering all functions and edge cases
